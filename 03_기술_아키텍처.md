# 03. 기술 스택 및 아키텍처

## 3-1. 기술 스택
| 영역 | 선정 | 이유 |
|---|---|---|
| 프레임워크 | Next.js 14 (App Router) | SSR/CSR, Vercel 최적 |
| 애니메이션 | Framer Motion + CSS sprite | 도트 이펙트 |
| 상태관리 | Zustand + TanStack Query | 캐릭터/EXP vs 서버 캐싱 |
| 스타일 | Tailwind CSS + 픽셀 토큰 | 8px 그리드 |
| 백엔드 | Supabase (Postgres/Auth/Realtime/Storage) | 올인원 |
| 사운드 | Howler.js | BGM/효과음 |

## 3-2. DB 설계
→ 첨부된 `supabase_schema.sql` 파일을 SQL Editor에 붙여넣어 즉시 생성.
**핵심:** 반복 일정은 `schedules.recur_rule`(RRULE)만 저장, 완료는 `schedule_logs`에 날짜별 인스턴스 기록.

## 3-3. 기능 구현 로직
- **구글 OAuth:** Supabase Google Provider → `signInWithOAuth({provider:'google'})`, 가입 트리거로 profiles 자동 생성.
- **반복 로직:** RRULE 저장 + 클라이언트 `rrule.js`로 조회범위 전개, log 없으면 미완료 표시. 대량 알림은 pg_cron + Edge Function.
- **EXP 산정:** 서버 RPC `complete_schedule()`로 트랜잭션 처리(조작 방지). `exp_gain = base * difficulty`, 레벨업 임계치 초과 시 level++/이월.
- **실시간:** co_quests, friendships만 Realtime 구독. 개인 일정은 TanStack Query 캐싱.

## 3-4. 성능 최적화
- 스프라이트 시트 + CSS `background-position`
- 화면 밖 에셋 Lazy Loading(`next/image`)
- PNG-8 인덱스 컬러, 애니메이션은 CSS `steps()`
- 일정 100개+ 대비 가상화(react-window) 또는 월 페이지네이션
- Storage CDN `Cache-Control: immutable`
